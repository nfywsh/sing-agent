# 歌声合成 Agent 设计与实现

## 1. 概述

### 1.1 项目目标

构建一个可集成到语音聊天的歌声合成 Agent，实现：

1. **音色设计**: 基于 VoiceDesign 生成参考音色
2. **歌曲生成**: 基于 ACE-Step 的歌词 → 歌曲合成
3. **实时转换**: 基于 SoulX-Singer 的音色转换

### 1.2 核心价值

| 特性 | 说明 |
|------|------|
| 延迟 | 实时响应 < 5秒 (SoulX-Singer 单独) |
| 个性化 | 用户可使用自己的音色唱歌 |
| 歌词保真 | ACE-Step 保留原始歌词，SoulX-Singer 不重写歌词 |

### 1.3 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                     歌声合成 Pipeline                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────────┐    ┌──────────────┐    ┌──────────────┐ │
│   │  VoiceDesign  │───→│   ACE-Step   │───→│  Demucs      │ │
│   │   生成音色    │    │   生成歌曲    │    │  人声分离    │ │
│   └──────────────┘    └──────────────┘    └──────────────┘ │
│         │                    │                   │         │
│         └────────────────────┴───────────────────┘         │
│                           │                                │
│                           ▼                                │
│                  ┌──────────────────┐                      │
│                  │  SoulX-Singer   │                      │
│                  │  音色转换(~3s)   │                      │
│                  └──────────────────┘                      │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 关键技术结论

#### SoulX-Singer 采样率

SoulX-Singer 内部使用 **24kHz** 采样率（配置: `soulxsinger.yaml`）。

**重要**: 最初文档记载使用 16kHz，但经过实际测试发现：
- VoiceDesign 输出 24kHz → 被重采样为 16kHz → SoulX-Singer 内部又升回 24kHz
- **双重重采样**导致音频模糊/蒙蒙的

**解决方案**: 使用原生 24kHz 采样率，避免重采样。

| 问题 | 原因 | 解决 |
|------|------|------|
| 音频模糊 | 24kHz→16kHz→24kHz 双重重采样 | 使用 24kHz 原生采样率 |
| 歌词被改 | SoulX-Singer 无歌词注入接口 | 使用更长的参考音色 + 低cfg |

## 2. 模型说明

### 2.1 模型列表

| 模型 | 端口 | 功能 | 采样率 | 延迟 |
|------|------|------|--------|------|
| **VoiceDesign** | 8020 | 音色生成 | 24kHz | ~5s |
| **ACE-Step** | 8001 | 歌曲生成 | 44.1kHz | ~70s |
| **Demucs** | 内置 | 人声分离 | 保持原采样率 | ~10s |
| **SoulX-Singer** | 7861 | 音色转换 | 24kHz原生 | ~3s |

### 2.2 模型特性详解

#### VoiceDesign (音色设计)

```
输入: 文本描述 (instructions) + 文本内容 (input)
输出: WAV 音频 (24kHz, 单声道)
延迟: ~5s
特点: 生成的音色特征与文本内容相关，文本越长音色特征越丰富
```

#### ACE-Step (歌曲生成)

```
输入: 歌词 (带结构标签 [verse][chorus] 等)
输出: MP3 音频 (44.1kHz, 立体声)
延迟: ~70s (30秒歌曲)
特点: 使用 /release_task 保留原始歌词，不重写
```

#### Demucs (人声分离)

```
输入: 任意格式 WAV/MP3
输出:
  - vocal.wav: 人声
  - no_vocals.wav: 伴奏
延迟: ~10s (与音频时长相关)
```

#### SoulX-Singer (音色转换)

```
输入:
  - prompt_audio: 参考音色音频 (24kHz 单声道 WAV)
  - target_audio: 待转换音频 (24kHz 单声道 WAV)
输出: 转换后的 WAV 音频
延迟: ~3s (与音频时长无关)

注意:
  - SoulX-Singer 内部使用 24kHz 采样率
  - 使用原生 24kHz 避免重采样导致的音质损失
  - cfg=1.0 可减少失真，保持歌词保真度
  - 没有歌词注入接口，歌词内容靠 target_audio 隐式保留
```

## 3. 方案设计

### 3.1 方案A: DiffRhythm → 人声分离 → SoulX-Singer

```
用户输入歌词 ──→ DiffRhythm 生成完整歌曲 ──→ 人声分离提取干声 ──→ SoulX-Singer 换音色 ──→ 输出
     │                 ~54s                     ~10s                   ~3s                   ~67s总延迟
     │
     └── 预生成模式: 后台批量生成，存储干声备用
```

### 3.2 工作流状态机

```
                    ┌─────────────┐
                    │   IDLE      │
                    └──────┬──────┘
                           │ 收到歌词
                           ▼
                 ┌─────────────────────┐
      ┌─────────│  PREGENERATING      │─────────┐
      │         │ DiffRhythm + 分离    │         │
      │         └─────────────────────┘         │
      │                    │ 完成后              │
      │                    ▼                    │
      │          ┌─────────────────┐             │
      │          │  STORAGE_READY  │             │
      │          │  歌曲已存储     │             │
      │          └────────┬────────┘             │
      │                   │                     │
      │                   │ 用户选择 + 提供音色  │
      │                   ▼                     │
      │         ┌─────────────────┐             │
      └────────→│  REAL_TIME_CONV │←────────────┘
                │ SoulX-Singer 转换│  新用户请求
                └────────┬────────┘
                         │ 3秒后完成
                         ▼
                ┌─────────────────┐
                │   COMPLETED     │
                │ 返回音频给用户   │
                └────────┬────────┘
                         │
                         │ 继续等待新请求
                         ▼
                ┌─────────────────┐
                │  STORAGE_READY  │
                └─────────────────┘
```

### 3.3 存储结构

```
/data/script/voice/sing-agent/
├── pregenerated/                    # 预生成歌曲库
│   ├── {song_id_1}/
│   │   ├── song.wav               # 原始完整歌曲
│   │   ├── vocal.wav              # 分离后干声
│   │   ├── vocal_f0.npy           # F0特征 (可选缓存)
│   │   └── metadata.json          # 元数据
│   ├── {song_id_2}/
│   │   └── ...
│   └── ...
├── cache/                           # 缓存
│   ├── prompts/                    # 用户提供的参考音色
│   │   └── {user_id}_{song_id}.wav
│   └── results/                    # SoulX-Singer 输出
│       └── {user_id}_{song_id}_converted.wav
├── logs/                           # 日志
│   └── sing-agent.log
└── doc/                            # 文档
    └── DESIGN.md
```
